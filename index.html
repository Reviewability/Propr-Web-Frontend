Your workflow, your rules

  • We aren't intrusive

    We don’t touch your code and only post a feedback link when pull requests are closed. Your workflow is unaffected.

  • We value your privacy

    We don’t store any personal information such as your email address or name and all feedback is collected anonymously.

  • Private repo? No issues

    Only collaborators whom you have rated would be able to see their pull requests. You choose how much data you share.

How does it work?

Can this be used as a platform to anonymously provide feedback on how the contributors could improve their pull requests? Yes, but there is so much more to it!

When one of the project maintainers close a pull request in a project that uses propr, our bot comments on the pull request with a feedback link. The short feedback survey has two open ended questions and three rating bars, all of which are optional. Filling the form is pretty much the last thing project managers and integrators have to do.
The reports dashboard collates feedback from integrators and project managers over multiple pull requests and a range of projects. Contributors to projects that use propr can readily see how their pull requests are doing, and how they can improve them. This is our way of ensuring that every feedback reaches the contributors!

About us

We are a group of researchers from the Software Engineering Research Group (SERG) at the Delft University of Technology. We set out to find out what factors contribute to the “reviewability” of pull requests submitted to open source projects and in the process ended up creating propr.
